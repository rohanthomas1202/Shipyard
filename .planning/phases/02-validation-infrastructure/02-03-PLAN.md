<objective>
Implement circuit breaker to stop retrying identical validation errors.

Purpose: Currently the agent retries up to 3 times on any error, wasting tokens and time when the same error repeats identically. After 2 identical errors on the same file/step, the agent should skip the step or escalate to a higher model tier instead of retrying blindly.

Output: Circuit breaker in should_continue() that detects repeated identical errors and routes to skip/advance.
</objective>

   ```python
   import re

   def _normalize_error(msg: str) -> str:
       """Strip line/col numbers and paths for error dedup comparison."""
       normalized = re.sub(r'[Ll]ine \d+', 'Line N', msg)
       normalized = re.sub(r'col \d+|column \d+|offset \d+', 'col N', normalized)
       normalized = re.sub(r'/[^\s:]+/', '/', normalized)
       return normalized.strip()
   ```

2. In `agent/graph.py`, add `_has_repeated_error(state: dict, threshold: int = 2) -> bool`:
   ```python
   def _has_repeated_error(state: dict, threshold: int = 2) -> bool:
       """Check if the same normalized error has occurred threshold+ times for current step."""
       history = state.get("validation_error_history", [])
       current_step = state.get("current_step", 0)
       lve = state.get("last_validation_error")
       if not lve:
           return False

       current_normalized = _normalize_error(lve.get("error_message", ""))
       count = 0
       for entry in history:
           if entry.get("step") == current_step and entry.get("normalized_error") == current_normalized:
               count += 1
       return count >= threshold
   ```
