**2. Effective Prompts and Interactions**
Document 4-5 effective prompt patterns with examples:
- Structured instruction decomposition: "Break this into steps with target_files, kind, and acceptance_criteria" -- produces PlanStep JSON the agent can execute
- Error-contextualized retries: Including specific anchor text, similarity score, and file state in retry prompts cut retry loops from 3 to 1 on average
- Graduated complexity for rebuild: Starting with simple field additions before multi-file coordination -- each success builds on prior context
- Context injection pattern: Providing schemas/specs as structured JSON in the instruction payload vs embedding in the instruction text
- Negative constraints: "Do NOT use unified diff format" type constraints in system prompts prevented common failure modes

**3. Code Analysis**
Analyze the Shipyard codebase with specifics:
- Total modules: ~30 Python files across agent/, server/, store/ + React frontend
- Key patterns: LangGraph StateGraph with conditional edges, anchor-based surgical editing, tiered model routing
- Code quality: Fully typed Python 3.11+ (str | None syntax), Pydantic models for persistence, TypedDict for graph state
- Test coverage: Integration tests with mock router fixtures, Ship-like fixture project
- Dependency management: Minimal -- 12 direct Python deps, no framework bloat
- Architecture strength: Clean separation between graph nodes, tools, and server layer
- Architecture weakness: Some duplication (_normalize_error in two files) due to circular import avoidance

**4. Strengths and Limitations of AI-Assisted Development**

Strengths:
- Consistency across long sessions (no fatigue, no context drift within a plan)
- Speed: 18 plans averaging ~3 min each = features that would take hours manually
- Structured approach: GSD framework prevented scope creep and maintained quality
- Test-first thinking: Verification criteria defined before implementation

Limitations:
- Context window pressure: Complex plans with many files degrade quality past 50% context usage
- Stale assumptions: AI occasionally references APIs that have changed since training
- Over-engineering tendency: AI adds unnecessary abstractions unless constrained
- Debugging blind spots: Cannot actually run and observe the app -- relies on test output

5-7 concrete takeaways:

- Error feedback loops matter more than retry count: Informed retries > blind retries
- Anchor-based editing is robust but brittle on large files: 200+ line files need line-range reads
