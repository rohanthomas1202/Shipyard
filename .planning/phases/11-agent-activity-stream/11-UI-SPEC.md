# Phase 11 — UI Design Contract

> Visual and interaction contract for the real-time agent activity stream: event timeline, auto-scroll behavior, new-event badge, and run sections. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon-to-label gaps inside event cards, inline badge padding |
| sm | 8px | Vertical gap between event cards, internal card padding |
| md | 16px | Panel body padding (left/right/top), run section header padding |
| lg | 24px | Vertical gap between run sections |
| xl | 32px | Not used in Phase 11 |
| 2xl | 48px | Not used in Phase 11 |
| 3xl | 64px | Not used in Phase 11 |

Exceptions:
- Scroll-bottom threshold: 50px from bottom edge triggers sticky-bottom auto-scroll mode. Not on 4px grid; tuned for scroll UX.
- New-event badge: 12px vertical padding, 16px horizontal padding for comfortable click target.

Source: Phase 8 UI-SPEC spacing scale (unchanged). Usage column updated for Phase 11 scope.

Accent (`#6366F1`) reserved for (Phase 11 scope):
- New-event badge background and pulse animation ring
- Active node indicator dot in the stream
- "planning" event type badge background tint
- Scroll-to-bottom badge click affordance

Never use accent for: event card backgrounds, timestamps, generic text, borders at rest.

```
+--------------------------------------------------+

|                                                    |
| RunSection: "Run: Optimize query..."               |

|                                                    |
+--------------------------------------------------+
|  [ 3 new events ]  <-- NewEventBadge (floating)   |

| Status bar: dot + "Agent Working - editing"        |
+--------------------------------------------------+
```

### Stream Container

- Full height of panel minus header (inherited from Phase 8) minus status bar (36px).
- `overflow-y: auto` with custom scrollbar (4px, from glass.css).
- Padding: 16px horizontal, 8px vertical (top of scroll area).

### Event Card Layout

```
+----------------------------------------------+
| [TypeBadge] [node label]        [timestamp]   |
| [body content - varies by type]               |
| [expandable detail - collapsed by default]    |
+----------------------------------------------+
```

- Card: `border-radius: var(--radius-element)` (8px), padding 12px.
- Gap between cards: 8px vertical.
- TypeBadge: 8px vertical padding, 8px horizontal, border-radius 4px, 11px font, semibold.

### NewEventBadge Layout

- Position: `absolute`, bottom 16px from status bar, centered horizontally in panel.
- Height: 32px, border-radius 16px (pill shape).
- Text: 12px, semibold, white on accent background.
- Shadow: `0 4px 12px rgba(99, 102, 241, 0.3)`.
- Z-index: 10 (above scroll content, below modal overlays).

### Stream Token Handling

- `stream` events accumulate tokens into the most recent StreamingBlock.
- New `stream` events with a different `node` or after a non-stream event create a new StreamingBlock.
- StreamingBlock uses a `ref` to append text (no re-render per token). Same pattern as existing `StreamingText`.
- When streaming ends (next non-stream event arrives), StreamingBlock freezes and becomes a static EventCard.

### Expandable Detail

- Default: collapsed, showing a "Show details" text link in muted color.
- Expanded: slides open with 200ms ease-out transition, max-height animation.
- Content: code-font block with `rgba(0, 0, 0, 0.2)` background, 8px padding, 4px border-radius.

## Interaction States

### Scroll Behavior

| State | Behavior |
|-------|----------|
| User at bottom (within 50px) | Auto-scroll enabled. New events smoothly scroll into view (`scrollTo` with `behavior: 'smooth'`). |
| User scrolls up (more than 50px from bottom) | Auto-scroll disabled. NewEventBadge appears with count of events received since scroll-up. |
| User clicks NewEventBadge | Scroll to bottom with `behavior: 'smooth'`. Re-enable auto-scroll. Badge disappears. |
| New run starts | Clear badge count. Scroll to bottom. Re-enable auto-scroll. |

Implementation: `onScroll` handler checks `scrollHeight - scrollTop - clientHeight <= 50`. Use `requestAnimationFrame` guard to avoid excessive checks (same pattern as Phase 10 diff scroll sync).

### Event Animation

| Trigger | Animation |
|---------|-----------|
| New event enters (auto-scroll active) | Slide up from bottom: `transform: translateY(8px)` to `translateY(0)`, `opacity: 0` to `1`, 200ms ease-out |
| New event enters (scrolled up) | No animation (event is off-screen; badge count increments) |
| Badge appears | Fade in: `opacity: 0` to `1`, 150ms ease-out, then pulse ring animation |
| Badge pulse | Repeating ring: `box-shadow` expands from 0 to 8px spread in accent color at 20% opacity, 2s interval |
| Expand detail | `max-height: 0` to `max-height: 300px`, 200ms ease-out. Overflow hidden during transition. |
| Collapse detail | `max-height: 300px` to `max-height: 0`, 150ms ease-in |

CSS keyframes to add to `glass.css`:

```css
@keyframes slide-up {
  from { transform: translateY(8px); opacity: 0; }
  to { transform: translateY(0); opacity: 1; }
}

@keyframes badge-pulse {
  0%, 100% { box-shadow: 0 0 0 0 rgba(99, 102, 241, 0.4); }
  50% { box-shadow: 0 0 0 8px rgba(99, 102, 241, 0); }
}
```

### Run Sections

| State | Visual |
|-------|--------|
| Single run (current) | No section header needed; events render directly |
| Multiple runs | Each run gets a sticky section header: instruction text (truncated) + relative timestamp |
| New run starts | Horizontal divider + new section header. Previous run section remains scrollable above. |
| Empty panel (no runs) | Empty state: icon + heading + body text, vertically centered |

### Idle vs Active States

| State | What the panel shows |
|-------|---------------------|
| Idle, no runs ever | Empty state (icon + "No activity yet" + body text) + AutonomyToggle |
| Idle, past runs exist | Last run's events visible in stream, scrollable |
| Active run | StepTimeline at top + live event stream with auto-scroll |
| Error during run | Error EventCard appears in stream + run status in status bar turns error color |

## Accessibility

| Requirement | Implementation |
|-------------|---------------|
| Scroll region | `role="log"` and `aria-live="polite"` on the stream container so screen readers announce new events |
| Event cards | Each card has `role="article"` with `aria-label` combining type + timestamp |
| New-event badge | `role="status"` and `aria-live="assertive"` so screen readers announce unread count changes |
| Badge click | `<button>` element with `aria-label="{N} new events, click to scroll to latest"` |
| Expand/collapse | `<button>` with `aria-expanded="true/false"` and `aria-controls` pointing to detail region ID |
| Reduced motion | All slide-up and pulse animations wrapped in `@media (prefers-reduced-motion: no-preference)`. With reduced motion: instant append, no pulse. |
| Color contrast | Badge: white (#FFFFFF) on accent (#6366F1) = 4.6:1 (WCAG AA). Event text: same contrast ratios as Phase 8 (verified). |
| Keyboard | Tab through event cards, Enter to expand/collapse details, badge focusable via Tab |
