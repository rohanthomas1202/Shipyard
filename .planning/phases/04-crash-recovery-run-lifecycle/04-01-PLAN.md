<objective>
Wire LangGraph AsyncSqliteSaver checkpointing into the agent graph so runs persist state at every node boundary and can resume after server crashes.

Purpose: Runs currently lose all progress on server crash or restart. This plan makes runs crash-recoverable by persisting checkpoint state to a dedicated SQLite DB and resuming interrupted runs on startup.

Output: `build_graph()` accepts a checkpointer, server lifespan manages AsyncSqliteSaver lifecycle, `thread_id` is passed in every ainvoke config, and interrupted runs resume automatically on restart.
</objective>

    2. Add `_resume_from_checkpoint(run_id: str)` async function:

       - Wraps in try/except, marks run as "error" on failure
