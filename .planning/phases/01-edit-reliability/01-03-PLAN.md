<objective>
Wire error feedback into retry prompts, add file freshness checking, migrate editor to structured outputs, and enhance validator error reporting.

Purpose: Currently, when edits fail and retry, the editor sees the exact same prompt with no information about what went wrong. The validator produces opaque error strings. This plan closes the feedback loop: failed edit diagnostics (anchor, best match, score) flow into the retry prompt, the editor uses structured LLM outputs (no more JSON parse failures), file freshness is checked before edits, and the validator produces structured error details.

Output: Enhanced editor_node with structured outputs + error feedback + freshness, enhanced validator_node with detailed errors, updated prompt template.
</objective>

    def _build_error_feedback(state: dict) -> str:
        """Build error feedback section for retry prompts."""
        error = state.get("error_state")
        if not error:
            return ""

        # Extract structured error info from edit_history
        edit_history = state.get("edit_history", [])
        last_error_entry = None
        for entry in reversed(edit_history):
            if entry.get("error"):
                last_error_entry = entry
                break

        if not last_error_entry:
            # Fallback: just include the error string
            return "PREVIOUS ATTEMPT FAILED:\n- Error: " + str(error)

        return ERROR_FEEDBACK_TEMPLATE.format(
            failed_anchor=last_error_entry.get("failed_anchor", "unknown")[:500],
            error_message=str(last_error_entry.get("error", error)),
            best_score=str(last_error_entry.get("best_score", "N/A")),
            best_match=last_error_entry.get("best_match_preview", "N/A")[:500],
        )
