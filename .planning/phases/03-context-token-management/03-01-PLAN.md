<objective>
Add token usage capture to the LLM layer and router, with per-run cost estimation.

Purpose: LIFE-02 requires each LLM call to track input/output tokens and aggregate cost per run. This plan modifies the LLM wrapper to return usage alongside content, adds a TokenTracker for cost estimation, and wires accumulation into the router -- all without changing the router's external return types.

Output: LLMResult/LLMStructuredResult dataclasses, TokenTracker module, updated router with usage accumulation, passing tests.
</objective>
