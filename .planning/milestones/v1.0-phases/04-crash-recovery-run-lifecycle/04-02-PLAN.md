<objective>
Add graceful cancellation so users can stop a running agent mid-execution with clean state rollback.

Purpose: Currently there is no way to stop a running agent. If a user submits a bad instruction or the agent goes off track, they must wait for completion or restart the server. This plan adds a cancel endpoint that cleanly stops execution and rolls back any in-progress edits.

Output: POST /runs/{run_id}/cancel endpoint, asyncio.Task storage in runs dict, CancelledError handling with edit rollback.
</objective>

       ```python
       @app.post("/runs/{run_id}/cancel")
       async def cancel_run(run_id: str):
           if run_id not in runs:
               raise HTTPException(status_code=404, detail="Run not found")
           run_entry = runs[run_id]
           if run_entry["status"] not in ("running",):
               raise HTTPException(status_code=409, detail=f"Cannot cancel run in state: {run_entry['status']}")
           run_entry["status"] = "cancelling"
           task = run_entry.get("task")
           if task and not task.done():
               task.cancel()
           return {"run_id": run_id, "status": "cancelling"}
       ```
