<objective>
Build the side-by-side diff view with real line-level diffing using jsdiff, Shiki syntax highlighting on both sides, and auto-open diff tabs when the agent proposes edits.

Purpose: The current DiffViewer uses a naive diff (all old lines red, all new lines green). This plan replaces it with a proper side-by-side diff using jsdiff's structuredPatch algorithm, showing context lines and correct change identification. This is the core review surface for agent edits.

Output: SideBySideDiff component with jsdiff + Shiki, updated DiffHeader with Accept Edit/Reject Edit + confirmation, EditorArea wired for diff tabs, auto-open on edit proposals.
</objective>

    **Step 2: Create web/src/components/editor/SideBySideDiff.tsx**

    This is the core component. It takes an `Edit` object and renders a two-column diff.

    Props:
    ```typescript
    import { structuredPatch } from 'diff'
    import { useState, useEffect, useRef, useCallback } from 'react'
    import { highlightCode } from '../../lib/shiki'
    import { api } from '../../lib/api'
    import { DiffHeader } from './DiffHeader'
    import type { Edit } from '../../types'

    interface SideBySideDiffProps {
      edit: Edit
      onResolved?: () => void
    }
    ```

    Component logic:
    1. Use `structuredPatch` from jsdiff to compute the diff between `edit.old_content` and `edit.new_content`:
       ```typescript
       const patch = structuredPatch(
         edit.file_path, edit.file_path,
         edit.old_content || '', edit.new_content || '',
         '', '', { context: 3 }
       )
       ```

    2. Build left/right line arrays from `patch.hunks`. For each hunk:
       - Lines starting with ' ' (context) go into BOTH left and right arrays with `type: 'context'`
       - Lines starting with '-' (removed) go into left array with `type: 'remove'`, and a blank spacer goes into right array
       - Lines starting with '+' (added) go into right array with `type: 'add'`, and a blank spacer goes into left array
       - Between hunks, insert a HunkSeparator row in both arrays

    3. Detect file language from `edit.file_path` extension using the same `_EXT_TO_LANG` mapping or derive from the extension string (e.g., `.py` -> `python`, `.tsx` -> `tsx`).

    4. Use `highlightCode()` from `shiki.ts` (Plan 01) to highlight both old_content and new_content. Store highlighted HTML in state via useEffect. Parse the highlighted HTML to extract per-line spans.

    5. Render two side-by-side columns, each 50% width:
       - Left column header: subtle "Old" label in muted text
       - Right column header: subtle "New" label in muted text
       - Each column has a line number gutter (50px wide) and code content area
       - Code background: `rgba(15, 17, 26, 0.95)`
       - Removed lines (left): `rgba(239, 68, 68, 0.15)` background, left border `3px solid rgba(239, 68, 68, 0.8)`
       - Added lines (right): `rgba(16, 185, 129, 0.15)` background, left border `3px solid rgba(16, 185, 129, 0.8)`
       - Context lines: transparent background
       - Line numbers: `rgba(148, 163, 184, 0.4)` color, right border `1px solid var(--color-border)`
       - Font: `var(--font-code)`, 13px, line-height 1.7
       - Divider between columns: `1px solid var(--color-border)`

    6. Synchronized vertical scroll: use a single scroll container for both sides, OR use `onScroll` handler on one side to set `scrollTop` on the other via refs.

    7. HunkSeparator: full-width row with `rgba(99, 102, 241, 0.08)` background, centered text `@@ ... @@` in 11px muted color.

    8. Handle edge cases:
       - New file (`old_content` is null): Left side shows "New file" placeholder, right shows all green lines
       - Deleted file (`new_content` is null): Left shows all red lines, right shows "File deleted" placeholder

    9. Accept/Reject handlers:
       ```typescript
       const [loading, setLoading] = useState(false)

       const handleAccept = async () => {
         setLoading(true)
         try {
           const opId = `op_${edit.id}_approve`
           await api.patchEdit(edit.run_id, edit.id, 'approve', opId)
           onResolved?.()
         } finally { setLoading(false) }
       }

       const handleReject = async () => {
         setLoading(true)
         try {
           const opId = `op_${edit.id}_reject`
           await api.patchEdit(edit.run_id, edit.id, 'reject', opId)
           onResolved?.()
         } finally { setLoading(false) }
       }
       ```

    10. Render DiffHeader at top with `filePath={edit.file_path}`, `onAccept={handleAccept}`, `onReject={handleReject}`, `loading={loading}`.

    Update button labels per UI-SPEC:

    - Add inline reject confirmation: when "Reject Edit" clicked, button text changes to "Confirm Reject?" for 3 seconds. Second click within window executes rejection. Timeout reverts label.

    ```typescript
    const [confirmingReject, setConfirmingReject] = useState(false)
    const rejectTimerRef = useRef<ReturnType<typeof setTimeout>>()

    const handleRejectClick = () => {
      if (confirmingReject) {
        clearTimeout(rejectTimerRef.current)
        setConfirmingReject(false)
        onReject?.()
      } else {
        setConfirmingReject(true)
        rejectTimerRef.current = setTimeout(() => setConfirmingReject(false), 3000)
      }
    }

    // Cleanup timer on unmount
    useEffect(() => () => clearTimeout(rejectTimerRef.current), [])
    ```
