## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding, scrollbar width |
| sm | 8px | Compact element spacing, gap between metrics items |
| md | 16px | Default element spacing, panel padding (px-4) |

| xl | 32px | Layout gaps between major sections |

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#0F111A` / `--color-background` | Page background behind panels |
| Secondary (30%) | `rgba(30, 33, 43, 0.6)` / `--color-surface` | Glass panels, ProgressHeader background, heatmap container |
| Accent (10%) | `#6366F1` / `--color-primary` | Progress bar fill, active metric highlight, expand/collapse chevron on hover |
| Destructive | `#EF4444` / `--color-error` | Failed task count, heatmap high-failure cells, error trace icon |
| Success | `#10B981` / `--color-success` | Completed task count, CI pass rate when >= 80%, compatible contract badge |
| Warning | `#F59E0B` / `--color-warning` | CI pass rate when 50-79%, breaking change indicator |
| Muted | `#94A3B8` / `--color-muted` | Secondary labels, metric descriptions, collapsed state text |
| Text | `#F8FAFC` / `--color-text` | Primary text, metric numbers |

Accent reserved for: progress bar fill, expand/collapse chevron hover state, active coverage percentage highlight. Never used for body text or metric labels.

**Heatmap cell colors:**
- 0 failures: `transparent` (no background)
- 1-2 failures: `rgba(245, 158, 11, 0.15)` (warning at 15% opacity)
- 3+ failures: `rgba(239, 68, 68, 0.25)` (error at 25% opacity)

All colors from existing `glass.css` tokens. No new color values introduced.

## Component Inventory

### 1. ProgressHeader (new component)

**Location:** `web/src/components/agent/ProgressHeader.tsx`
**Placement:** Inside AgentPanel, between StepTimeline and stream body (before the `flex-1 overflow-hidden` div). Renders above the activity stream.
**Behavior:** Collapsible. Default state: expanded. Click toggles collapsed/expanded. State stored in local `useState`.

**Expanded layout:**
```
[Progress bar ===========================-------  72%]  12/17 tasks
Coverage: 72%    CI Pass: 91%    Failed: 2    Running: 3
```

- Row 1: Full-width progress bar (6px height, rounded-full) + completion fraction label right-aligned
- Row 2: 4 metric items in a horizontal grid (`grid grid-cols-4 gap-2`), each showing label + value

**Collapsed layout:**
```
[Progress bar ===========================-------  72%]  12/17 tasks
```

- Row 1 only. Row 2 hidden.

**Progress bar styling:**
- Track: `rgba(255, 255, 255, 0.06)` background, `h-1.5`, `rounded-full`
- Fill: `var(--color-primary)`, `rounded-full`, `transition-all duration-300`
- Width: `${metrics.coveragePct}%`

**Metric item styling:**
- Label: 12px, `--color-muted`, weight 500
- Value: 20px, `--color-text`, weight 600
- Failed count value uses `--color-error` when > 0
- CI pass rate value uses `--color-success` when >= 80%, `--color-warning` when 50-79%, `--color-error` when < 50%

**Border:** Bottom border `1px solid var(--color-border)` separating from activity stream.

**Empty state:** Component returns `null` when `progressMetrics` is `null` in the Zustand store (no DAG running).

**Interaction:** Click anywhere on the header toggles expanded/collapsed. Cursor: `pointer`. Chevron icon (`expand_more` / `expand_less`) at far right of Row 1, 16px, `--color-muted`, hover: `--color-primary`.

### 2. FailureHeatmap (new component)

**Location:** `web/src/components/agent/FailureHeatmap.tsx`
**Placement:** Not embedded in AgentPanel. Accessed via a future debug/observability view. For Phase 14, this is primarily a data model with a minimal table rendering.

**Layout:** HTML table with modules as rows and error categories (syntax, test, contract, structural) as columns.

**Table styling:**
- Container: `glass-panel` class, `p-4`, `rounded-lg`
- Header row: 12px, `--color-muted`, weight 600, uppercase
- Cell: 32px x 32px min, centered number, 12px weight 500
- Cell background: color-coded by failure count (see heatmap cell colors above)
- Module name column: 14px, `--color-text`, weight 400, left-aligned, max-width 200px, truncate with ellipsis
- Grid lines: `1px solid var(--color-border)`

**Empty state:** "No failure data" centered in the table area, 12px, `--color-muted`.

### 3. DecisionTrace (new component)

**Location:** `web/src/components/agent/DecisionTrace.tsx`
**Placement:** Rendered inline within failed task event cards in the activity stream. Expandable detail section.

**Collapsed state:** Single line showing error category badge + truncated error message (max 80 chars).
**Expanded state:** Shows full error message, files read list, and truncated LLM context (prompt first 200 chars, response first 200 chars).

**Error category badge:**
- Pill shape: `px-2 py-0.5 rounded-full text-xs font-medium`
- Colors by category:
  - syntax: `rgba(239, 68, 68, 0.15)` bg, `--color-error` text
  - test: `rgba(245, 158, 11, 0.15)` bg, `--color-warning` text
  - contract: `rgba(99, 102, 241, 0.15)` bg, `--color-primary` text
  - structural: `rgba(148, 163, 184, 0.15)` bg, `--color-muted` text

**LLM context block:**
- Background: `rgba(15, 17, 26, 0.5)` (darker than surface)
- Font: JetBrains Mono, 12px, `--color-muted`
- Border-left: `2px solid var(--color-border)`
- Padding: 8px 12px
- Label above: "LLM Prompt" / "LLM Response" in 11px uppercase `--color-muted`

**Files read list:**
- Bullet list, 12px, `--color-text`, `font-code`
- Max 5 files shown, "+N more" link if exceeds

**Toggle:** Click on the collapsed row to expand. Chevron icon rotates 90deg on expand.

## Interaction Contracts

### ProgressHeader Collapse/Expand
- **Trigger:** Click anywhere on ProgressHeader area
- **Animation:** `transition-all duration-200 ease-in-out` on the metrics grid height (0 to auto via max-height trick: `max-height: 0` collapsed, `max-height: 80px` expanded, `overflow: hidden`)
- **Chevron rotation:** `transition-transform duration-200`, rotates 180deg
- **Default state:** Expanded
- **Persistence:** None (resets on page reload)

### DecisionTrace Expand
- **Trigger:** Click on the collapsed trace row
- **Animation:** Same max-height transition pattern
- **Default state:** Collapsed
- **Keyboard:** Not required for Phase 14 (data inspection, not primary workflow)

### Progress Bar Animation
- **Fill transition:** `transition-all duration-300 ease-out` on width changes
- **No pulse/glow animation** — clean, static fill that smoothly grows

### Heatmap Cell Hover
- **Trigger:** Mouse hover on heatmap cell
- **Effect:** `opacity: 0.8` on the cell, tooltip showing "module: N failures (category)" — use `title` attribute, no custom tooltip component needed for Phase 14

## Accessibility Notes

- ProgressHeader: `role="region"` with `aria-label="DAG progress metrics"`
- ProgressHeader toggle: `aria-expanded="true|false"` on the clickable container
- Progress bar: `role="progressbar"` with `aria-valuenow`, `aria-valuemin="0"`, `aria-valuemax="100"`
- FailureHeatmap table: standard `<table>` with `<thead>` and `<th scope="col|row">`
- DecisionTrace: `aria-expanded` on the expandable row
- Activity stream already has `role="log"` and `aria-live="polite"` — ProgressHeader sits outside this region
