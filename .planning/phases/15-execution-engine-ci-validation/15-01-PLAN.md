    Create `agent/orchestrator/branch_manager.py` with class `BranchManager`:

    Module docstring: "Git branch lifecycle management for isolated task execution."

    Constructor: `__init__(self, project_path: str)` storing `self._cwd = project_path` and `self._lock = asyncio.Lock()`.

    Private helper: `async def _git(self, *args: str) -> dict` that runs `["git", *args]` via `run_command_async(cwd=self._cwd, timeout=30)`. Logs warning on non-zero exit_code. Does NOT acquire lock (callers do).

    Public methods (all acquire self._lock via `async with self._lock:`):

    1. `async def create_and_checkout(self, branch: str, base: str = "main") -> bool`:
       - checkout base
       - `git branch -D {branch}` (ignore failure -- branch may not exist)
       - `git checkout -b {branch}`
       - return exit_code == 0

    2. `async def rebase_and_merge(self, branch: str) -> bool`:
       - `git checkout {branch}`
       - `git rebase main`
       - if rebase fails: `git rebase --abort`, return False
       - `git checkout main`
       - `git merge --ff-only {branch}`
       - return merge exit_code == 0

    3. `async def cleanup(self, branch: str) -> None`:
       - `git checkout main`
       - `git branch -D {branch}` (force delete, ignore failure)

    4. `async def verify_branch(self) -> str`:
       - `git rev-parse --abbrev-ref HEAD`
       - return stdout.strip()

    Per D-09, only fast-forward merges. Per D-03, rebase failure = conflict = return False (caller requeues). Per research pitfall 1, asyncio.Lock serializes all git mutations. Per research pitfall 3, cleanup always runs (called from scheduler finally block).
