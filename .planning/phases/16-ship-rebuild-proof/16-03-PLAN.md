```python
@pytest.fixture
async def auth_token(client):
    """Register + login, return auth token for authenticated tests."""
    # Try signup first (may already exist)
    await client.post("/api/auth/signup", json={"email": TEST_EMAIL, "password": TEST_PASSWORD})
    # Login to get token
    resp = await client.post("/api/auth/login", json={"email": TEST_EMAIL, "password": TEST_PASSWORD})
    assert resp.status_code in (200, 201), f"Login failed: {resp.status_code} {resp.text}"
    data = resp.json()
    token = data.get("token") or data.get("accessToken") or data.get("access_token", "")
    return token
```

Tests (all `@pytest.mark.asyncio`):

1. `test_health_endpoint(client)` -- GET `/health` returns 200, response has a body
2. `test_auth_signup(client)` -- POST `/api/auth/signup` with unique email returns 200 or 201, response JSON has "user" or "token" key
3. `test_auth_login(client)` -- POST `/api/auth/login` with TEST_EMAIL/TEST_PASSWORD returns 200, response JSON has token
4. `test_auth_invalid_login(client)` -- POST `/api/auth/login` with wrong password returns 401 or 400
5. `test_create_item(client, auth_token)` -- POST `/api/documents` (or `/api/items`) with `{"title": "Smoke Test Item", "content": "test"}` and `Authorization: Bearer {auth_token}` header returns 200 or 201, response JSON has "id"
6. `test_list_items(client, auth_token)` -- GET `/api/documents` with auth header returns 200, response is list (or has "data" key that is list)
7. `test_get_single_item(client, auth_token)` -- Creates an item, then GET `/api/documents/{id}` returns 200 with matching id
8. `test_delete_item(client, auth_token)` -- Creates an item, DELETE `/api/documents/{id}` returns 200 or 204
9. `test_deployed_url()` -- If `SHIP_DEPLOYED_URL` env var is set, GET that URL returns 200. Skip if not set: `pytest.skip("SHIP_DEPLOYED_URL not set")`
10. `test_routes_summary(client)` -- GET `/health`, POST `/api/auth/signup`, POST `/api/auth/login`, GET `/api/documents`. Print summary: `f"All {N} API routes responding correctly"` (per UI-SPEC copy). Assert at least 3 of 4 return non-5xx status codes.

Each test includes meaningful assertion messages, e.g. `assert resp.status_code == 200, f"Expected 200, got {resp.status_code}: {resp.text[:200]}"`.
