# Phase 8 — UI Design Contract

> Visual and interaction contract for the IDE shell foundation: resizable three-panel layout, top bar with instruction input, and Zustand state architecture. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none (custom glassmorphic system) |
| Preset | not applicable |
| Component library | none (custom components with Tailwind CSS 4) |
| Icon library | Material Symbols (Google Fonts CDN, already loaded) |
| Font | Plus Jakarta Sans (UI), JetBrains Mono (code) |

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding, resize handle hit area inset |
| sm | 8px | Compact element spacing, panel header internal padding |
| md | 16px | Default element spacing, panel body padding |
| lg | 24px | Section padding, top bar vertical padding |
| xl | 32px | Layout gaps between panels (outer shell padding) |
| 2xl | 48px | Major section breaks (not used in Phase 8) |
| 3xl | 64px | Page-level spacing (not used in Phase 8) |

Exceptions:
- Panel resize handle hit target: 12px wide (not on 4px grid) for comfortable drag affordance. Visual handle line is 4px wide centered within the 12px hit area.
- Top bar height: 52px fixed (not on 8-point grid; optimized for single-line input + vertical padding).

## Typography

| Role | Size | Weight | Line Height | Font |
|------|------|--------|-------------|------|
| Body | 14px | 400 (regular) | 1.5 | Plus Jakarta Sans |
| Label | 11px | 600 (semibold) | 1.2 | Plus Jakarta Sans |
| Heading | 16px | 600 (semibold) | 1.2 | Plus Jakarta Sans |
| Code | 13px | 400 (regular) | 1.6 | JetBrains Mono |

Notes:
- Label role is used for panel header titles (uppercase, tracking-wider: `letter-spacing: 0.05em`).
- Body role is used for instruction input text, dropdown items, status text.
- Heading role is used for the top bar project name and section headers.
- Code role is used in any code-displaying area within panels (Phase 8 scope: status messages that show file paths).

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#0F111A` | Page background behind mesh gradient |
| Secondary (30%) | `rgba(30, 33, 43, 0.6)` + `backdrop-filter: blur(24px)` | Glass panels (file explorer, editor area, agent stream), top bar |
| Accent (10%) | `#6366F1` (Indigo) | See reserved list below |
| Destructive | `#EF4444` | Destructive actions only (none in Phase 8) |
| Success | `#10B981` | Run status: completed |
| Warning | `#F59E0B` | Run status: running/in-progress |
| Muted text | `#94A3B8` | Secondary labels, panel headers, placeholder text |
| Border | `rgba(255, 255, 255, 0.08)` | Panel borders, divider lines, input borders |

Accent (`#6366F1`) reserved for:
- "Run" / submit instruction button (primary CTA)
- Active/focused input border highlight
- Active panel resize handle on drag
- Run status indicator dot when status is `waiting_for_human`
- Floating restore-panel button background (when a panel is collapsed)

Never use accent for: panel backgrounds, text, borders at rest, icons at rest.

### Panel-Specific Surface Colors

| Element | Background | Border |
|---------|-----------|--------|
| Top bar | `rgba(30, 33, 43, 0.8)` + blur(24px) | Bottom: `rgba(255, 255, 255, 0.08)` |
| Left panel (file explorer) | `rgba(30, 33, 43, 0.6)` + blur(24px) | Standard glass-panel |
| Center panel (editor area) | `rgba(30, 33, 43, 0.4)` + blur(24px) | Standard glass-panel |
| Right panel (agent stream) | `rgba(30, 33, 43, 0.6)` + blur(24px) | Standard glass-panel |
| Resize handle (rest) | transparent | none |
| Resize handle (hover/drag) | `rgba(99, 102, 241, 0.4)` | none |

Center panel uses lower opacity (0.4) so the mesh gradient shows through more, creating visual depth hierarchy: top bar (most opaque) > side panels > center.

## Layout Specification

### Overall Structure

```
+---------------------------------------------------------------+
| TopBar (52px fixed height)                                    |
| [ProjectSelector] [InstructionInput............] [RunStatus]  |
+---------------------------------------------------------------+
|            |  handle  |                |  handle  |            |
|  Explorer  |    ||    |   Editor Area  |    ||    |   Agent    |
|  (left)    |    ||    |   (center)     |    ||    |  (right)   |
|            |    ||    |                |    ||    |            |
+---------------------------------------------------------------+
```

### Panel Sizing

| Panel | Default Size | Min Size | Collapsed Size |
|-------|-------------|----------|----------------|
| Left (explorer) | 20% | 180px | 48px (icon rail) |
| Center (editor) | 50% | 300px | N/A (cannot collapse) |
| Right (agent) | 30% | 200px | 0px (fully hidden) |

- `autoSaveId: "shipyard-ide-panels"` for localStorage persistence.
- Default sizes are percentages of remaining width after top bar renders.
- Center panel is not collapsible. It absorbs space when side panels collapse.

### TopBar Layout

```
+---------------------------------------------------------------+
| [ProjectSelector ▼] | [instruction input...] [Run ▶] | ● Status |
+---------------------------------------------------------------+
```

- Left zone (flex-shrink-0): ProjectSelector dropdown, max-width 200px.
- Center zone (flex-1): InstructionInput. Compact mode: single-line, 40px height. Expanded mode: multi-line textarea, max 120px height, triggered on focus/click.
- Right zone (flex-shrink-0): RunStatusIndicator (dot + label), gap-sm from input.

### InstructionInput Behavior

| State | Height | Appearance |
|-------|--------|------------|
| Compact (blurred, empty) | 36px | Single-line input, muted border, placeholder visible |
| Compact (blurred, has text) | 36px | Single-line input showing truncated text |
| Expanded (focused) | 36px-120px (auto-grows) | Multi-line textarea, accent border, full placeholder, submit button visible |

- Submit on: Click "Run instruction" button OR `Cmd+Enter` / `Ctrl+Enter`.
- After submit: Input clears, collapses back to compact mode.
- Disabled state: While a run is active (`running` or `waiting_for_human`), input is disabled with 50% opacity. Tooltip: "Wait for the current run to finish."

### Panel Collapse/Expand

| Trigger | Action |
|---------|--------|
| Click collapse icon in PanelHeader | Collapse panel to min size |
| Double-click resize handle | Toggle collapse/expand |
| Click expand icon (in collapsed rail) | Expand panel to last saved size |
| Keyboard: `Cmd+B` | Toggle left panel |
| Keyboard: `Cmd+J` | Toggle right panel |

Collapse animation: `react-resizable-panels` handles the size transition. No custom animation needed.

### Resize Handle Styling

| State | Visual |
|-------|--------|
| Rest | Invisible (transparent, 12px hit area) |
| Hover | 2px wide vertical line, `rgba(99, 102, 241, 0.4)`, centered in hit area |
| Dragging | 2px wide vertical line, `#6366F1` (full accent), centered in hit area |
| Cursor | `col-resize` on hover and drag |

## State Architecture Contract

### Zustand Store: `wsStore`

Replaces WebSocket event distribution from `WebSocketContext`. Connection management stays in a hook/ref.

```typescript
interface WSStore {
  // Connection
  status: 'connecting' | 'connected' | 'disconnected'
  setStatus: (s: WSStore['status']) => void

  // Run snapshot (replaces Context snapshot)
  snapshot: RunSnapshot | null
  setSnapshot: (s: RunSnapshot | null) => void

  // Agent events (consumed by ActivityStream panel only)
  agentEvents: WSEvent[]
  appendAgentEvent: (e: WSEvent) => void
  clearAgentEvents: () => void

  // File change tracking (consumed by FileExplorer panel only)
  changedFiles: Record<string, 'modified' | 'added' | 'deleted'>
  setFileChanged: (path: string, status: 'modified' | 'added' | 'deleted') => void
  clearChangedFiles: () => void

  // Last sequence per run
  lastSeq: Record<string, number>
  setLastSeq: (runId: string, seq: number) => void
}
```

Selector contract (prevents render storms):
- ActivityStream: `useWsStore(s => s.agentEvents)`
- FileExplorer: `useWsStore(s => s.changedFiles)`
- TopBar/RunStatus: `useWsStore(s => s.snapshot)`
- Connection banner: `useWsStore(s => s.status)`

### Zustand Store: `workspaceStore`

Client-side UI state for tabs, selection, layout.

```typescript
interface WorkspaceStore {
  // Tab state
  openTabs: Tab[]
  activeTabId: string | null
  openFile: (path: string, pin?: boolean) => void
  openDiff: (editId: string) => void
  closeTab: (tabId: string) => void
  setActiveTab: (tabId: string) => void

  // File explorer state
  selectedPath: string | null
  setSelectedPath: (path: string | null) => void
}
```

### React Context: `ProjectContext` (kept, not migrated)

Low-frequency state. No changes to interface in Phase 8:
- `projects`, `currentProject`, `selectProject`, `submitInstruction`, `currentRun`, `runs`

## Interaction States

### Top Bar

| State | Visual |
|-------|--------|
| No project selected | ProjectSelector shows "Select project...", instruction input disabled |
| Project selected, idle | ProjectSelector shows project name, instruction input enabled |
| Run active | Instruction input disabled (50% opacity), RunStatus shows "Running..." with animated dot |
| Run waiting for review | Instruction input disabled, RunStatus shows "Waiting for review" with pulsing accent dot |
| Run completed | Instruction input re-enabled, RunStatus shows "Completed" with success dot |
| WebSocket disconnected | Slim warning banner below top bar: "Connection lost. Reconnecting..." |

### Panel Resize

| State | Visual |
|-------|--------|
| Default | Three panels visible at saved (or default) sizes |
| Dragging handle | Handle turns accent color, panels resize in real-time |
| Left collapsed | Left panel shrinks to 48px icon rail (folder + settings icons) |
| Right collapsed | Right panel shrinks to 0px, floating accent button appears bottom-right to restore |
| Both collapsed | Center panel fills available width |

## Accessibility

| Requirement | Implementation |
|-------------|---------------|
| Panel resize keyboard | `react-resizable-panels` provides arrow key resize on focused handle (built-in WAI-ARIA) |
| Focus management | `Cmd+B` focuses left panel, `Cmd+J` focuses right panel |
| Instruction input | `Cmd+Shift+P` focuses instruction input (existing hotkey, kept) |
| Color contrast | All text on glass surfaces meets WCAG AA: `#F8FAFC` on `rgba(30,33,43,0.6)` = contrast ~11:1; `#94A3B8` on same = contrast ~5.5:1 |
| Screen reader | Panel headers use `role="heading" aria-level="2"`, resize handles have `aria-label="Resize {panel name}"` |
| Reduced motion | Resize transitions respect `prefers-reduced-motion: reduce` (disable 0.2s transitions) |
