   - In `_syntax_check()`, add a `.py` branch BEFORE the final `return {"valid": True, "error": None}` fallthrough:
     ```python
     if ext == ".py":
         try:
             with open(file_path, "r", encoding="utf-8", errors="replace") as f:
                 source = f.read()
             ast.parse(source, filename=file_path)
             return {"valid": True, "error": None}
         except SyntaxError as e:
             error = f"Line {e.lineno}, col {e.offset}: {e.msg}"
             if e.text:
                 error += f" | {e.text.strip()}"
             return {"valid": False, "error": error}
     ```

  <action>
1. In `agent/nodes/validator.py`, in `validator_node()`, harden the LSP try/except block:
   - The existing `except Exception as e:` already catches broad failures. Enhance it to also catch `asyncio.TimeoutError` explicitly (in case it is not a subclass of Exception in the runtime).
   - Add a timeout wrapper around the `_lsp_validate` call: `result = await asyncio.wait_for(_lsp_validate(client, file_path, last_edit), timeout=30.0)`. This ensures LSP validation cannot hang indefinitely even if the LSP client's internal timeout fails.
   - Catch `asyncio.TimeoutError` in the except block alongside `Exception`:
     ```python
     try:
         result = await asyncio.wait_for(
             _lsp_validate(client, file_path, last_edit),
             timeout=30.0,
         )
         if result is not None:
             return result
     except (Exception, asyncio.TimeoutError) as e:
         tracer.log("validator", {
             "file": file_path,
             "lsp_error": str(e),
             "lsp_error_type": type(e).__name__,
             "fallback": "syntax_check",
         })
     ```

2. In `_lsp_validate()`, add defensive checks:
   - Wrap `client.get_diagnostics()` calls in try/except to catch connection errors. If either baseline or post diagnostics fail, return `None` (fall back to syntax check) rather than propagating the error.
   - After `diff_diagnostics()`, if `new_errors` is empty, explicitly return `{"error_state": None}` (already done, just verify).

   - Use `unittest.mock.AsyncMock` for the LSP mocks.
