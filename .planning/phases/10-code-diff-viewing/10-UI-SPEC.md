### Diff-Specific Colors

These colors are tuned for visibility against the glassmorphic translucent background (Pitfall 11 from PITFALLS.md -- higher opacity required):

| Element | Background | Text | Left Border |
|---------|-----------|------|-------------|
| Added line | rgba(16, 185, 129, 0.15) | var(--color-text) | rgba(16, 185, 129, 0.8) 3px solid |
| Removed line | rgba(239, 68, 68, 0.15) | rgba(239, 68, 68, 0.9) | rgba(239, 68, 68, 0.8) 3px solid |
| Context line | transparent | var(--color-text) | none |
| Hunk separator | rgba(99, 102, 241, 0.08) | var(--color-muted) | none |

Source: Existing DiffLine.tsx colors preserved. Hunk separator is new (uses accent at low opacity).

Accent reserved for: Accept button background, active tab bottom border (2px), welcome tab icon highlight, diff header icon background.

### Code Area Background

The code viewing area (both file viewer and diff sides) uses a solid background `rgba(15, 17, 26, 0.95)` instead of the translucent glass panel background. This ensures syntax highlighting colors remain readable against a near-opaque dark surface while the surrounding chrome stays glassmorphic.

## Interaction Contracts

### Tab Bar (D-06, D-07)

| Interaction | Behavior |
|-------------|----------|
| Single-click file in explorer | Opens preview tab (italic label, replaces existing preview tab). If file already open, activates it. |
| Double-click file in explorer | Opens pinned tab (normal label, persists). If preview tab exists for this file, pins it. |
| Click tab | Activates that tab, shows its content in the editor area |
| Click tab close button (X) | Closes tab. If active tab closed, activates the previous tab. If no tabs remain, shows WelcomeTab. |
| Middle-click tab | Closes that tab (same as close button) |
| Tab overflow (>N tabs) | Tab strip scrolls horizontally. No wrapping. Scroll with mouse wheel or drag. |
| Diff auto-open (D-11) | When agent proposes an edit, a pinned diff tab auto-opens and activates. Does not replace preview tabs. |

### Tab Visual States

| State | Label Style | Background | Indicator |
|-------|-------------|------------|-----------|
| Active + pinned | 13px regular, var(--color-text) | rgba(30, 33, 43, 0.8) | 2px bottom border var(--color-primary) |
| Active + preview | 13px italic, var(--color-text) | rgba(30, 33, 43, 0.8) | 2px bottom border var(--color-primary) |
| Inactive + pinned | 13px regular, var(--color-muted) | transparent | none |
| Inactive + preview | 13px italic, var(--color-muted) | transparent | none |
| Hover (any) | unchanged label | rgba(255, 255, 255, 0.03) | close button appears (16px X icon) |

### Tab Icons

| Tab Type | Icon | Size |
|----------|------|------|
| File tab | `description` (Material Symbols) | 16px |
| Diff tab | `compare_arrows` (Material Symbols) | 16px |
| Welcome tab | `code` (Material Symbols) | 16px |

### File Viewer (D-08)

| Interaction | Behavior |
|-------------|----------|
| Open file tab | Fetch content via `api.readFile(path)`. Show skeleton loader during fetch. Render with Shiki highlighting. |
| Scroll | Native vertical scroll within the code area. Horizontal scroll for long lines (no wrapping). |
| Text selection | Allowed (read-only, but user can copy code) |
| Binary file (D-09) | Centered placeholder: `description` icon at 48px + "Binary file -- cannot display" in 14px muted text |
| Empty file (D-09) | Centered placeholder: `draft` icon at 48px + "Empty file" in 14px muted text |
| Large file (>2000 lines) | Shiki highlighting runs async via useEffect. Plain text shown first, then swapped to highlighted version. |

### File Viewer Layout

```
+--------------------------------------------------+
| [line gutter 50px] | [code content, full width]   |
| 1                  | import { useState } from ... |
| 2                  | import { api } from ...      |
| 3                  |                              |
+--------------------------------------------------+
```

- Line gutter: 50px wide, right-aligned numbers, `rgba(148, 163, 184, 0.4)` color, right border `1px solid var(--color-border)`
- Code area: left padding 16px, `white-space: pre`, horizontal overflow scroll
- Background: `rgba(15, 17, 26, 0.95)` (near-opaque dark)

### Side-by-Side Diff (D-01, D-02, D-03, D-04)

| Interaction | Behavior |
|-------------|----------|
| View diff | Two equal-width columns. Left = old content, right = new content. Both Shiki-highlighted. |
| Context lines (D-04) | 3 unchanged lines above and below each change hunk. Collapsed hunks between changes show HunkSeparator. |
| Scroll | Both sides scroll together (synchronized vertical scroll). Horizontal scroll is independent per side. |
| Accept edit | Click "Accept" in DiffHeader. Calls `api.patchEdit(runId, editId, 'approve', opId)`. Button shows spinner during request. |
| Reject edit | Click "Reject" in DiffHeader. Calls `api.patchEdit(runId, editId, 'reject', opId)`. Button shows spinner during request. |
| After accept/reject | Tab label updates to show status. If more pending edits, next diff tab auto-activates. |
| New file (old_content is null) | Left side shows empty with "New file" label. Right side shows full new content, all lines green. |
| Deleted file (new_content is null) | Left side shows full old content, all lines red. Right side shows empty with "File deleted" label. |

### Side-by-Side Diff Layout

```
+---------------------------------------------------+
| DiffHeader: filepath | step label | [Reject][Accept] |
+---------------------------------------------------+
| OLD (left side)           | NEW (right side)        |
| [gutter] [code]           | [gutter] [code]         |
|  10  context line          |  10  context line        |
|  11  context line          |  11  context line        |
|  12  context line          |  12  context line        |
|  13- removed line          |                          |
|  14- removed line          |  13+ added line          |
|                            |  14+ added line          |
|                            |  15+ added line          |
|  15  context line          |  16  context line        |
|  16  context line          |  17  context line        |
|  17  context line          |  18  context line        |
| --- @@ collapsed hunks @@ ---|--- @@ collapsed hunks @@ ---|
|  45  context line          |  47  context line        |
+---------------------------------------------------+
```

- Each side: 50% width of the editor area
- Each side has its own line number gutter (50px) and code column
- Divider between sides: 1px solid `var(--color-border)`
- Removed lines: full-row background `rgba(239, 68, 68, 0.15)` on left side only
- Added lines: full-row background `rgba(16, 185, 129, 0.15)` on right side only
- Hunk separator: full-width row, `rgba(99, 102, 241, 0.08)` background, centered text "@@ lines N-M @@" in muted color

### Welcome Tab (D-10)

Shown when no tabs are open. Vertically and horizontally centered in the editor area.

| Element | Spec |
|---------|------|
| Icon | `code` Material Symbol, 48px, var(--color-primary) at 0.6 opacity |
| Heading | "No files open" -- 20px semibold var(--color-text) |
| Body | "Select a file from the explorer to view its contents, or wait for the agent to propose edits." -- 14px regular var(--color-muted) |
| Max width | 360px, centered |

### Loading States

| Context | Skeleton |
|---------|----------|
| File content loading | 8 lines of animated shimmer bars (height 13px, varying widths 40-90%), glassmorphic background. Gutter shows line numbers 1-8 in muted color. |
| Diff computing | Same shimmer skeleton on both sides of the diff layout |
| Tab switching | Instant -- no loading state. Content is rendered from cache if previously loaded. Uncached tabs show skeleton. |

### Error States

| Context | Display |
|---------|---------|
| File fetch failed | Centered: `error` icon at 48px in var(--color-error) + "Failed to load file" heading + "The file may have been moved or deleted. Try refreshing the explorer." body text + "Retry" button (ghost style) |
| Diff fetch failed | Same pattern: "Failed to load diff" + "The edit record may no longer be available." + "Retry" button |

## Shiki Configuration

| Property | Value |
|----------|-------|
| Theme | `vitesse-dark` (dark theme that pairs well with midnight backgrounds and glassmorphic aesthetics) |
| Loading strategy | Async via `createHighlighter()` in useEffect. Show plain text immediately, swap to highlighted on ready. |
| Bundle strategy | Dynamic grammar loading -- only fetch grammar for detected language. Code-split highlighter via `React.lazy()`. |
| Language detection | Use `FileContent.language` from API response. Map to Shiki language identifier. Fallback: `text` for unknown. |
| Rendering | Use `codeToHast()` + `hast-util-to-jsx-runtime` for React elements. Never use `dangerouslySetInnerHTML`. |
| Large files | Files >2000 lines: highlight visible viewport only. Show plain text for off-screen content. |

## Accessibility Notes

| Concern | Approach |
|---------|----------|
| Tab close button | 28px hit area minimum, `aria-label="Close {filename}"` |
| Active tab | `aria-selected="true"`, `role="tab"` on each tab, `role="tablist"` on TabBar, `role="tabpanel"` on content area |
| Diff color contrast | Line type is communicated by both color AND left-border indicator (not color-only) |
| Keyboard tab navigation | Left/Right arrow keys switch between tabs when tab bar is focused. Delete/Backspace closes active tab. |
| Code block | `role="code"` on the pre element |
