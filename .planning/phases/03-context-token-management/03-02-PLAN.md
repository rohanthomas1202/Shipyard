    - reader_node loads files over 200 lines as skeleton view

    1. Modify agent/nodes/reader.py:
       - Import content_hash from agent.tools.file_ops
       - Add helper function _read_with_range(file_path: str, start: int | None = None, end: int | None = None) -> str:
         * Read all lines from file
         * If start/end provided: return lines[start:end] with prefix "[Lines {start+1}-{min(end, len(lines))} of {len(lines)}]\n"
         * If no range and len(lines) > 200: return skeleton view -- lines[:30] + ["\n... ({len(lines) - 40} lines omitted) ...\n\n"] + lines[-10:], prefixed with "[{len(lines)} lines total -- skeleton view]\n"
         * Otherwise: return full content "".join(lines)
       - Update the file reading loop in reader_node:
         * Extract optional line_range from step_entry dict: line_range = step_entry.get("line_range") -- expects [start, end] or None
         * If line_range is a list of 2 ints, pass start=line_range[0], end=line_range[1] to _read_with_range
         * Otherwise pass None, None (triggers auto skeleton for large files)
         * Store content via _read_with_range instead of raw open().read()
         * Compute and store file_hashes[fpath] = content_hash(raw_content) where raw_content is the FULL file (for freshness, not the skeleton)
       - Return both file_buffer and file_hashes from reader_node

       - test_large_file_skeleton: create 300-line file, call _read_with_range(path), assert "[300 lines total -- skeleton view]" in result, assert "lines omitted" in result, assert first line present, assert last line present
       - test_explicit_line_range: create 100-line file, call _read_with_range(path, 20, 40), assert "[Lines 21-40 of 100]" in result, assert line 21 content present, assert line 1 NOT present
       - test_reader_node_small_file: mock state with a small file, call reader_node, assert full content in file_buffer
       - test_reader_node_large_file_skeleton: mock state with 300-line file, call reader_node, assert skeleton markers in file_buffer
       - test_reader_node_line_range_from_step: mock state with plan step containing "line_range": [10, 30], assert only that range loaded
       - test_reader_node_returns_file_hashes: call reader_node, assert file_hashes key present in return dict with content_hash value
