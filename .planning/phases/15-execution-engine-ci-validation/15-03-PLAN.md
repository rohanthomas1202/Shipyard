    Module docstring: "Context pack assembly for scoped agent execution."

    Constant: `MAX_CONTEXT_FILES = 5`

    Dataclass `ContextPack`:
    ```python
    @dataclass
    class ContextPack:
        task_id: str
        primary_files: list[str]       # From task metadata (planner-selected)
        dependency_files: list[str]    # From import graph analysis
        contracts: list[str]           # From task.contract_inputs
        recent_changes: list[str]      # Files changed by recently-merged tasks

        @property
        def all_files(self) -> list[str]:
            """Deduplicated file list, capped at MAX_CONTEXT_FILES. Priority: primary > deps > contracts."""
            seen: set[str] = set()
            result: list[str] = []
            for f in self.primary_files + self.dependency_files + self.contracts:
                if f not in seen:
                    seen.add(f)
                    result.append(f)
            return result[:MAX_CONTEXT_FILES]
    ```

    Function `parse_python_imports(file_path: str, project_path: str) -> list[str]`:
    - Read file content, find `from X import Y` and `import X` lines using regex
    - Convert module paths to file paths (e.g., `agent.tools.shell` -> `agent/tools/shell.py`)
    - Return list of resolved file paths that exist on disk
    - Handle FileNotFoundError gracefully (return empty list)

    Class `ContextPackAssembler`:
    - `__init__(self, project_path: str)` storing path
    - `def assemble(self, task: TaskNode, recent_changes: list[str] | None = None) -> ContextPack`:
      - Extract primary_files from `task.metadata.get("target_files", [])`
      - Extract contracts from `task.contract_inputs`
      - For each primary file, call `parse_python_imports` to get dependency_files
      - Deduplicate dependency_files, exclude primary_files
      - Return ContextPack with all fields populated

    Per D-04 (hybrid assembly: planner picks primary, analyzer adds transitive). The import parser is a simplified stand-in since the full analyzer module from Phase 13 may not have a module map available at this point.

    Tests in `tests/test_context_packs.py`: Test ContextPack deduplication and capping. Test assemble with mock TaskNode metadata. Test parse_python_imports with a temporary Python file containing imports. Test graceful handling of missing files.

    ```python
    @dataclass
    class OwnershipViolation:
        file_path: str
        owning_module: str | None
        task_module: str
        reason: str
    ```

    Function `build_ownership_map(tasks: list[TaskNode]) -> dict[str, str]`:
    - For each task, extract target_files from task.metadata.get("target_files", [])
    - Map each file to the task's module (from task.metadata.get("module", task.label))
    - Return dict mapping file_path -> module_name
    - Later files overwrite earlier (last writer wins for shared files)

    Class `OwnershipValidator`:
    - `__init__(self, project_path: str, ownership_map: dict[str, str])` storing both
    - `async def validate(self, task_id: str, task_module: str) -> list[OwnershipViolation]`:
      - Run `git diff --name-only main...HEAD` via run_command_async to get changed files
      - For each changed file: look up in ownership_map
      - If owner exists and owner != task_module: create OwnershipViolation
      - If file not in map: skip (unowned files are OK -- config, shared utils)
      - Return list of violations (empty = all good)

    Per D-05 (soft enforcement via validator). Agent writes freely, validator checks after. Violations are returned to the scheduler which treats them as task failure (the task made unauthorized changes).
