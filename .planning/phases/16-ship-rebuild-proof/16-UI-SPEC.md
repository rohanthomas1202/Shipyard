## Phase Classification

**This is a backend integration phase, not a frontend phase.** Phase 16 exercises the autonomous pipeline (Phases 12-15) to rebuild Ship from scratch and deploy it. The UI contract is intentionally minimal and covers two areas:

1. **Rebuilt Ship app** -- what the Playwright E2E tests must verify renders correctly
2. **Demo presentation** -- what "Green CI + live URL" means visually

No new Shipyard UI components, styles, or design tokens are introduced.

## Rebuilt Ship -- Playwright E2E Visual Expectations

These are the visual assertions the Playwright E2E tests must make about the **rebuilt Ship app** (not Shipyard). Per D-08 and D-09:

### Page Load (SHIP-03)
- Page loads at the public URL without blank screen
- `document.title` is non-empty
- `body` element is visible
- No `pageerror` events (critical JS errors) during load
- At least one interactive element (button, link, or input) is visible

### Auth Flow (SHIP-02)
- `/signup` or `/register` page renders a form with email and password fields
- Form submission navigates away from signup page (to dashboard, home, or login)
- `/login` page renders a form with email and password fields
- Successful login navigates to an authenticated page

### CRUD Flow (SHIP-02)
- Authenticated user can see a list/table of content items (or empty state with create prompt)
- Create action produces a new item visible in the list
- Edit action updates an existing item's displayed content
- Delete action removes an item from the list

### Navigation (SHIP-02)
- Primary navigation is visible (nav bar, sidebar, or menu)
- Navigation links route to distinct pages without full page reload
- Back button returns to previous page

### Visual Smoke (SHIP-03)
- No layout overflow causing horizontal scroll on viewport 1280x720
- Text is readable (not white-on-white or black-on-black)
- Interactive elements have visible boundaries or affordances
